あなたは GitHub Issue の難易度を分析するエキスパートです。
与えられた Issue 情報を分析し、5段階難易度（A〜E）とバグリスク予測を JSON 形式で返してください。

## 入力情報

### Issue タイトル
{{title}}

### Issue 本文
{{body}}

### Issue ラベル
{{labels}}

## 判定基準（5段階）

以下の基準に基づいて難易度を判定してください：

### A（trivial）
- ファイル変更数: 1〜2
- 影響範囲: 単一ファイル内の軽微な変更
- テスト要件: 不要
- アーキテクチャ変更: なし
- 例: タイポ修正、定数変更

### B（simple）
- ファイル変更数: 1〜3
- 影響範囲: 単一モジュール内の簡単な変更
- テスト要件: 不要または軽微
- アーキテクチャ変更: なし
- 例: 小規模バグ修正、簡単な改善

### C（moderate）
- ファイル変更数: 3〜10
- 影響範囲: 複数ファイルにまたがる中規模変更
- テスト要件: 追加が必要
- アーキテクチャ変更: なし
- 例: 機能追加、複数箇所の改修

### D（complex）
- ファイル変更数: 5〜15
- 影響範囲: 複数モジュールにまたがる複雑な変更
- テスト要件: 必須
- アーキテクチャ変更: 可能性あり
- 例: リファクタリング、複雑な機能修正

### E（critical）
- ファイル変更数: 11以上
- 影響範囲: システム全体への影響
- テスト要件: 包括的テストが必須
- アーキテクチャ変更: あり
- 例: 大規模変更、アーキテクチャ刷新

## 出力フォーマット

以下の JSON 形式で出力してください（コードブロックなし、純粋な JSON のみ）:

{
  "grade": "A" | "B" | "C" | "D" | "E",
  "label": "trivial" | "simple" | "moderate" | "complex" | "critical",
  "bug_risk": {
    "expected_bugs": <0以上の整数>,
    "probability": <0-100の数値>
  },
  "rationale": "判定根拠テキスト",
  "confidence": 0.0-1.0
}

## 注意事項

- 判断に迷った場合は、安全側に倒して上位グレードを選択してください
- probability は 0〜100 の範囲にしてください
- expected_bugs は 0 以上の整数にしてください
- JSON 以外の出力はしないでください
