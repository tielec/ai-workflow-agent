## Guidelines by Issue Type

### bug
- Clearly describe reproduction steps
- Distinguish expected behavior vs actual behavior
- Include error handling and test cases in the fix approach

### task
- State background and purpose
- Describe implementation approach and impact scope
- Define concrete done criteria

### enhancement
- State objective and user story
- Describe design approach and technical considerations
- Evaluate backward compatibility impact

## Sub-Issue Body Structure

Include the following sections:
- ## Overview (explicitly mention relation to the parent issue)
- ## Current State (problem status and reproduction steps)
- ## Proposal (fix/implementation approach)
- ## Target Files (list of file paths to change)
- ## Implementation Hints (concrete hints and code snippets)
- ## Action Items (checklist)
- ## Done Criteria (include test requirements)
