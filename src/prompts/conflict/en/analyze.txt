# Merge Conflict Resolution Plan (Analyze)

**IMPORTANT: Write all document content in English. All sections, descriptions, and explanations must be in English.**

Analyze the merge conflict context below and output a JSON resolution plan.

## Target Conflict Files ({conflict_file_count} files)

You MUST provide exactly one resolution for each of the following files. No file may be omitted.

{conflict_file_list}

## Input (JSON)
The merge conflict context is provided in the following JSON file. Read this file using the Read tool to analyze: `{merge_context_file_path}`

## Rules
- Sensitive files (.env, *.pem, *.key, etc.) must be placed in `skippedFiles`
- `resolutions` MUST include resolution strategies for ALL of the target conflict files listed above. Missing even one file will result in an error
- `strategy` must be one of `both | ours | theirs | manual-merge`
- For `ours`, `theirs`, `both` strategies, `resolvedContent` can be an empty string `""` (the system resolves these mechanically)
- Only for `manual-merge` strategy, include the full resolved file content in `resolvedContent`
- Write the output as a JSON file to this exact absolute path using the Write tool: `{output_file_path}`
- Do NOT output JSON to stdout. Always write to the file

## Output Format
{
  "resolutions": [
    {
      "filePath": "src/example.ts",
      "strategy": "manual-merge",
      "resolvedContent": "...",
      "notes": "Reasoning or notes"
    }
  ],
  "skippedFiles": [".env"],
  "warnings": [".env requires manual resolution"]
}
