# マージコンフリクト解消計画（分析）

**重要: すべてのドキュメント内容を日本語で記述してください。すべてのセクション、説明、解説は日本語で書いてください。**

以下のマージコンフリクト文脈を分析し、解消計画をJSONで出力してください。

## 対象コンフリクトファイル（{conflict_file_count}個）

以下のすべてのファイルに対して、必ず1つずつresolutionを返してください。省略は不可です。

{conflict_file_list}

## 入力（JSON）
マージコンフリクトの文脈情報は以下のJSONファイルに記載されています。Read ツールでこのファイルを読み込んで分析してください: `{merge_context_file_path}`

## ルール
- 機密ファイル（.env, *.pem, *.key など）は `skippedFiles` に入れてください
- `resolutions` には上記の対象コンフリクトファイル**すべて**に対する解消戦略を記載してください。1つでも欠けている場合はエラーになります
- `strategy` は `both | ours | theirs | manual-merge`
- `ours`, `theirs`, `both` 戦略の場合、`resolvedContent` は空文字列 `""` で構いません（システムが自動で解消します）
- `manual-merge` 戦略の場合のみ、`resolvedContent` に解消後のファイル全体の内容を記載してください
- 出力は以下の絶対パスにJSONファイルとして Write ツールで書き出してください: `{output_file_path}`
- stdout への出力は不要です。必ずファイルに書き出してください

## 出力フォーマット
{
  "resolutions": [
    {
      "filePath": "src/example.ts",
      "strategy": "manual-merge",
      "resolvedContent": "...",
      "notes": "理由や注意点"
    }
  ],
  "skippedFiles": [".env"],
  "warnings": [".env は手動解消が必要です"]
}
