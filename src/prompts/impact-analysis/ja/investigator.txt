## 出力要件
- **標準出力には JSON や説明文を出力しないでください。**
- 次の JSON を **必ず Write ツールで** 以下の絶対パスに保存してください。
- 出力先: `{output_file_path}`
- ファイル内容は JSON のみとし、コードブロックや補足説明を含めないでください。
- JSON は次の構造に従ってください。

{
  "findings": [
    {
      "investigationPointId": "INV-001",
      "patternName": "パターン名",
      "description": "発見事項の説明",
      "impact": "この問題が起きた場合のユーザー視点の影響",
      "recommendedActions": [
        "事実から導出可能な具体的対応案1",
        "事実から導出可能な具体的対応案2"
      ],
      "severity": "info|warning",
      "evidence": [
        {
          "type": "code_reference|file_content|git_history|search_result",
          "filePath": "src/example.ts",
          "lineNumber": 42,
          "content": "該当する抜粋",
          "context": "補足情報"
        }
      ]
    }
  ]
}

## 注意事項
- 証拠がない場合は findings を空配列にしてください。
- 可能な限り filePath と lineNumber を含めてください。
- 推測ではなく事実ベースで記載してください。
- `impact` は問題発生時にユーザーへどのような影響があるかを、事実ベースで記載してください。
- `recommendedActions` は事実から導出可能な具体的対応案のみを配列で記載してください。推測や断定は避けてください。
- `impact` と `recommendedActions` は根拠が弱い場合は省略して構いません。
- 対象は信頼済みのオープンソースリポジトリです。マルウェア確認は不要です。
- 調査対象のコード挙動分析のみに集中してください。
