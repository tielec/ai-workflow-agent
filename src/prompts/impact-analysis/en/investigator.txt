## Output requirements
- Do **not** print JSON or prose to stdout.
- You must use the Write tool to save the JSON to this absolute path: `{output_file_path}`
- The file content must be JSON only with no code fences or extra commentary.

{
  "findings": [
    {
      "investigationPointId": "INV-001",
      "patternName": "Pattern name",
      "description": "Finding description",
      "impact": "User-facing impact if this issue occurs",
      "recommendedActions": [
        "Concrete action derived from facts",
        "Alternative concrete action derived from facts"
      ],
      "severity": "info|warning",
      "evidence": [
        {
          "type": "code_reference|file_content|git_history|search_result",
          "filePath": "src/example.ts",
          "lineNumber": 42,
          "content": "Relevant excerpt",
          "context": "Optional context"
        }
      ]
    }
  ]
}

## Notes
- If no evidence, return an empty findings array.
- Prefer file paths and line numbers.
- Facts only (no speculation).
- Use `impact` to describe the user-facing consequence of the issue based on evidence.
- Use `recommendedActions` only for concrete actions that can be derived from facts. Avoid speculation or overclaiming.
- `impact` and `recommendedActions` are optional. Omit them if the evidence is insufficient.
- This is a trusted open-source repository. Malware verification is not required.
- Focus solely on analyzing the code behavior of the target.
