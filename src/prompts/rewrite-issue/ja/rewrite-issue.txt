## 出力形式

以下のJSON形式で出力してください：

```json
{
  "title": "新しいタイトル（80文字以内）",
  "body": "新しい本文（Markdown形式）",
  "metrics": {
    "completeness": 85,
    "specificity": 70
  }
}
```

## 再設計のガイドライン

1. **タイトル**
   - 具体的で簡潔なタイトルにする
   - 影響範囲や変更内容が分かるようにする
   - 80文字以内に収める

2. **本文構成**
   - ## 概要: 課題の背景と目的を明確に
   - ## 現在の状況: 現状の問題点を具体的に
   - ## 提案内容: 解決策を詳細に
   - ## 対象ファイル: コード文脈から特定したファイルパス
   - ## 実装のヒント: 具体的な実装方針
   - ## アクションアイテム: チェックリスト形式のタスク

3. **具体性の確保**
   - ファイルパスは `src/commands/xxx.ts` のように具体的に
   - 関数名やクラス名を含める
   - 必要に応じて行番号も記載

4. **採点指標**
   - completeness: 必要なセクションの網羅率（0-100）
   - specificity: 具体的な情報の含有率（0-100）
